# Resume Application Value Model

This document records the product model for choosing truthful resume content
for a specific opportunity. It complements the decision process in
[`RESUME_DECISION_ENGINE.md`](RESUME_DECISION_ENGINE.md) and does not change
the current staged workflow.

## Relevance versus evidence strength

A directly relevant experience or project must not automatically outrank a
stronger transferable experience or project.

Selection must distinguish at least:

- direct role relevance
- transferable relevance
- technical complexity
- system scope
- integration depth
- candidate ownership
- leadership and initiative
- validation and testing depth
- demonstrated outcomes
- quantitative scope, when supported
- distinctiveness
- interview-story value
- redundancy
- line cost

Project or experience strength must be based on canonical profile evidence,
not vague model impressions. Observable strength can include:

- integrating several subsystems
- solving an ambiguous engineering problem
- owning architecture or major design decisions
- validating a system through structured testing
- demonstrating difficult technical execution
- coordinating hardware and software
- producing a supported measurable outcome
- leading meaningful technical work

A simple directly related project may be selected when it fills an important
role requirement, but it must compete against stronger transferable projects
using overall application value.

The planner’s objective is to build the strongest truthful argument for hiring
the candidate for this specific opportunity. It is not to select the content
with the greatest literal or topical overlap.

When the initial plan is underfilled, the planner must evaluate the strongest
remaining supported content, including:

- additional bullets for strong selected entries
- stronger transferable projects
- additional relevant experiences
- complementary leadership, testing, integration, or ownership evidence
- additional supported skills

It must never select weak or irrelevant material solely to occupy space.

Later planning diagnostics must expose, for every candidate:

- relevance classification
- strength dimensions
- estimated line cost
- selection status
- rejection reason
- marginal value compared with selected content
- exact stopping reason

Examples in product discussions are explanatory only. No company, job posting,
or fixture-specific example is a production-selection rule.
